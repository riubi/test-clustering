# [Clustering Physical Activity](https://www.kaggle.com/competitions/clustering-physical-activity)

Кластеризовать данные носимых устройств (PAMAP2) по типу физической активности:
три IMU (рука, грудь, лодыжка) и температурные датчики, разметки нет.
Метрика соревнования: accuracy.

**Ноутбук воспроизводит: accuracy 0.768 на лидерборде**. KMeans на
7 кластеров по оконным и частотным признакам, число кластеров подобрано по
лидерборду, метки сглажены по баутам. Выход: `submission.csv` (534601 строк,
`Index` с 0).

![Лидерборд](assets/leaderboard.png)

---

## Задача

По сигналам IMU без разметки сгруппировать строки по типу активности (лежать,
сидеть, ходьба, бег, велосипед, прыжки и т.д.). Один отсчёт активность почти не
различает, поэтому решает оконная статистика по восстановленному временному ряду.

---

## Данные

Положить CSV локально перед запуском:

```
data/monitoring_data.csv - 534601 строк, 53 столбца, разметки нет
```

Структура столбцов: `timestamp`, по 17 каналов на узел (температура,
ускорение Acc16 и Acc6, гироскоп, магнитометр, ориентация), `subject_id`.
Ориентация по документации PAMAP2 невалидна и отбрасывается, Acc6 дублирует Acc16.

---

## Решение

1. EDA: блочные пропуски (выпадение узла), дубль Acc6/Acc16, перемешанный
   порядок строк.
2. Восстановление ряда сортировкой по `(subject_id, timestamp)`, 100 Гц.
3. Feature engineering: оконные mean/std/min/max модулей ускорения и гироскопа
   (интенсивность), оконные средние осей акселерометра (поза), частотные
   признаки по модулю ускорения (каденс: доминирующая частота, энергия полосы,
   спектральная энтропия).
4. Проверка признаков на утечку испытуемого (subject NMI): температура
   отброшена как прокси личности. Дополнительно PCA и важность через RandomForest.
5. Подбор K: внутренние метрики плюс прямой подбор по лидерборду, выбран K=7.
6. Сравнение моделей: KMeans, MiniBatchKMeans, GaussianMixture, Birch,
   Agglomerative, DBSCAN.
7. Сглаживание меток по баутам (метка постоянна внутри активности) и сабмит
   с перенумерацией меток по порядку.

---

## Результаты

Число кластеров подобрано напрямую по лидерборду (внутренние метрики на этих
данных смещены к малым K и не равны accuracy):

| K | 3 | 4 | 5 | 6 | 7 | 8 | 12 | 18 |
|---|---|---|---|---|---|---|----|----|
| accuracy | 0.32 | 0.30 | 0.70 | 0.71 | **0.77** | 0.65 | 0.50 | 0.32 |

Пик на K=6-7, это реальное число различимых активностей. Что дало рост скора:

| Шаг | accuracy |
|---|---|
| KMeans K=12, мгновенные признаки | 0.49 |
| K=7 + оконные признаки + сглаживание по баутам | 0.71 |
| + частотные признаки (каденс) | **0.77** |

Признаки проверены на утечку испытуемого (subject NMI): температура её резко
повышает (кодирует человека), поэтому отброшена. Финал KMeans: лучший silhouette
среди моделей, низкий subject NMI, штатно размечает весь датасет.
